## Opprette et verkté…¶y

### Plassering

De kan defineres:

### Struktur

Den enkleste mæ°“ten æ°“ lage verkté…¶y pæ°“ er æ°“ bruke `tool()`-hjelperen som gir typesikkerhet og validering.

```ts title=".opencode/tools/database.ts" {1}

export default tool({
  description: "Query the project database",
  args: {
    query: tool.schema.string().describe("SQL query to execute"),
  },
  async execute(args) {
    // Your database logic here
    return `Executed query: ${args.query}`
  },
})
```

**filnavnet** blir **verkté…¶ynavnet**. Ovennevnte oppretter et `database`-verkté…¶y.

#### Flere verkté…¶y per fil

Du kan ogsæ°“ eksportere flere verkté…¶y fra en enkelt fil. Hver eksport blir **et separat verkté…¶y** med navnet **`<filename>_<exportname>`**:

```ts title=".opencode/tools/math.ts"
import { tool } from "@openagt/plugin"

export const add = tool({
  description: "Add two numbers",
  args: {
    a: tool.schema.number().describe("First number"),
    b: tool.schema.number().describe("Second number"),
  },
  async execute(args) {
    return args.a + args.b
  },
})

export const multiply = tool({
  description: "Multiply two numbers",
  args: {
    a: tool.schema.number().describe("First number"),
    b: tool.schema.number().describe("Second number"),
  },
  async execute(args) {
    return args.a * args.b
  },
})
```

Dette lager to verkté…¶y: `math_add` og `math_multiply`.

#### Navnekollisjoner med innebygde verkté…¶y

Egendefinerte verkté…¶y er né…¶klet etter verkté…¶ynavn. Hvis et egendefinert verkté…¶y bruker samme navn som et innebygd verkté…¶y, vil det egendefinerte verkté…¶yet ha forrang.

For eksempel erstatter denne filen det innebygde `bash`-verkté…¶yet:

```ts title=".opencode/tools/bash.ts"

  description: "Restricted bash wrapper",
  args: {
    command: tool.schema.string(),
  },
  async execute(args) {
    return `blocked: ${args.command}`
  },
})
```

:::note
Foretrekk unike navn med mindre du med vilje é…¶nsker æ°“ erstatte et innebygd verkté…¶y. Hvis du vil deaktivere et innebygd verkté…¶y, men ikke overstyre det, bruk [tillatelser](/docs/permissions).
:::

### Kontekst

Verkté…¶y mottar kontekst om gjeldende é…¶kt:

```ts title=".opencode/tools/project.ts" {8}
import { tool } from "@openagt/plugin"

export default tool({
  description: "Get project information",
  args: {},
  async execute(args, context) {
    // Access context information
    const { agent, sessionID, messageID, directory, worktree } = context
    return `Agent: ${agent}, Session: ${sessionID}, Message: ${messageID}, Directory: ${directory}, Worktree: ${worktree}`
  },
})
```

Bruk `context.directory` for é…¶ktens arbeidskatalog.
Bruk `context.worktree` for git-arbeidstreet.

### Skrive et verkté…¶y i Python

Du kan skrive verkté…¶yene dine pæ°“ hvilket som helst spræ°“k du vil. Her er et eksempel som legger til to tall ved hjelp av Python.

Fé…¶rst lager du verkté…¶yet som et Python-skript:

```python title=".opencode/tools/add.py"

```

Lag deretter verkté…¶ydefinisjonen som pæ°“kaller den:

```ts title=".opencode/tools/python-add.ts" {10}

import path from "path"

export default tool({
  description: "Add two numbers using Python",
  args: {
    a: tool.schema.number().describe("First number"),
    b: tool.schema.number().describe("Second number"),
  },
  async execute(args, context) {
    const script = path.join(context.worktree, ".opencode/tools/add.py")
    const result = await Bun.$`python3 ${script} ${args.a} ${args.b}`.text()

```

Her bruker vi [`Bun.$`](https://bun.com/docs/runtime/shell)-verkté…¶yet for æ°“ kjé…¶re Python-skriptet.
