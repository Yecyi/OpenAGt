## Strukturert Utdata

Du kan be om strukturert JSON-utdata fra modellen ved 氓 spesifisere et `format` med et JSON-skjema. Modellen vil bruke et `StructuredOutput`-verkt酶y for 氓 returnere validert JSON som samsvarer med skjemaet ditt.

### Grunnleggende bruk

```typescript

  path: { id: sessionId },
  body: {
    parts: [{ type: "text", text: "Research Anthropic and provide company info" }],
    format: {
      type: "json_schema",
      schema: {
        type: "object",
        properties: {
          company: { type: "string", description: "Company name" },
          founded: { type: "number", description: "Year founded" },
          products: {
            type: "array",
            items: { type: "string" },
            description: "Main products",
          },
        },
        required: ["company", "founded"],
      },
    },
  },
})

// Access the structured output
console.log(result.data.info.structured_output)
// { company: "Anthropic", founded: 2021, products: ["Claude", "Claude API"] }
```

### Utdataformattyper

| Type          | Beskrivelse                                                     |
| ------------- | --------------------------------------------------------------- |
| `text`        | Standard. Standard tekstrespons (ingen strukturert utdata)      |
| `json_schema` | Returnerer validert JSON som samsvarer med det angitte skjemaet |

### JSON Skjemaformat

N氓r du bruker `type: 'json_schema'`, oppgi:

| Felt         | Type            | Beskrivelse                                                |
| ------------ | --------------- | ---------------------------------------------------------- |
| `type`       | `'json_schema'` | P氓krevd. Spesifiserer JSON-skjemamodus                     |
| `schema`     | `object`        | P氓krevd. JSON Schema-objekt som definerer utdatastrukturen |
| `retryCount` | `number`        | Valgfritt. Antall valideringsfors酶k (standard: 2)          |

### Feilh氓ndtering

Hvis modellen ikke klarer 氓 produsere gyldig strukturert utdata etter alle fors酶k, vil svaret inkludere en `StructuredOutputError`:

```typescript

  console.error("Failed to produce structured output:", result.data.info.error.message)
  console.error("Attempts:", result.data.info.error.retries)
}
```

### Beste praksis

1. **Gi klare beskrivelser** i skjemaegenskapene dine for 氓 hjelpe modellen med 氓 forst氓 hvilke data som skal trekkes ut
2. **Bruk `required`** for 氓 spesifisere hvilke felt som m氓 v忙re til stede
3. **Hold skjemaer fokuserte** - komplekse n酶stede skjemaer kan v忙re vanskeligere for modellen 氓 fylle ut riktig
4. **Angi passende `retryCount`** - 酶k for komplekse skjemaer, reduser for enkle
