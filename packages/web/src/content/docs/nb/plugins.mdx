**npm-plugins** installeres automatisk ved hjelp av Bun ved oppstart. Pakker og deres avhengigheter er bufret i `~/.cache/opencode/node_modules/`.

**Lokale plugins** lastes direkte fra plugin-katalogen. For ć°“ bruke eksterne pakker, mć°“ du opprette en `package.json` i konfigurasjonskatalogen din (se [Dependencies](#dependencies)), eller publisere plugin-en til npm og [legg den til i konfigurasjonen din](/docs/config#plugins).

## Opprette en plugin

En plugin er en **JavaScript/TypeScript-modul** som eksporterer en eller flere plugin-moduler
funksjoner. Hver funksjon mottar et kontekstobjekt og returnerer et krokobjekt.

---

### Avhengigheter

Lokale plugins og tilpassede verkté…¶y kan bruke eksterne npm-pakker. Legg til en `package.json` til konfigurasjonskatalogen med avhengighetene du trenger.

```json title=".opencode/package.json"
{
  "dependencies": {
    "shescape": "^2.1.0"
  }
}
```

OpenAGt kjé…¶rer `bun install` ved oppstart for ć°“ installere disse. Programtilleggene og verkté…¶yene dine kan deretter importere dem.

```ts title=".opencode/plugins/my-plugin.ts"

export const MyPlugin = async (ctx) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
  }
}
```

### Grunnleggende struktur

```js title=".opencode/plugins/example.js"
export const MyPlugin = async ({ project, client, $, directory, worktree }) => {
  console.log("Plugin initialized!")

  return {
    // Hook implementations go here
  }
}
```

Plugin-funksjonen mottar:

- `project`: Gjeldende prosjektinformasjon.
- `directory`: Gjeldende arbeidskatalog.
- `worktree`: Git-arbeidstrebanen.
- `client`: En OpenAGt SDK klient for samhandling med AI.
- `$`: Buns [shell API](https://bun.com/docs/runtime/shell) for ć°“ utfé…¶re kommandoer.

### Egendefinerte verkté…¶y

Plugins kan ogsć°“ legge til egendefinerte verkté…¶y til OpenAGt:

  return {

### Komprimerings-hooks
