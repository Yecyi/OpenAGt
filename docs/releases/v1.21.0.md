## Highlights

OpenAGt v1.21.0 ships the **domain-specialized agentic helper** runtime: a new
critical-review pipeline (MPACR), a three-layer memory architecture (3LMA) that
accumulates domain knowledge across sessions, and a dynamic expert / prompt
template registry (DXR + PTS) that lets users extend the coordinator without
touching source code.

This release is the largest behavioral expansion since v1.16. All new
behaviour is **off by default** — existing v1.20 configurations behave
identically until the new feature flags are enabled. The 38 builtin coordinator
roles are unchanged and remain a closed enum; user experts attach via an
additive registry.

Test coverage for the affected subsystems went from ~30 % to **227 passing
tests** with zero typecheck errors across `packages/openagt`.

## Key Features

### MPACR — Multi-Perspective Adversarial Critical Review (Stream A)

A four-stage structured-debate replacement for the legacy single-pass
`reviseNode`. Behind `EffortProfile.mpacr_enabled`. When enabled, every
plan-/output-/handoff-/reducer-/verifier-revise gate that has a concrete
target artifact is expanded into a debate graph:

1. **Steel-man** — restate the artifact's strongest reading
2. **Red-team (parallel)** — K critics attack from preset perspectives
   (`factuality`, `coherence`, `risk`, `domain_expertise`, `user_value`)
3. **Defense** — owner rebuts or concedes each critique (silence forbidden)
4. **Synthesis** — emit a `CriticalReviewVerdict` with constructive feedback
5. **Calibration** — record posterior + Brier feedback for long-term tuning

- `CriticalReviewVerdict` schema gains `evidence_for`, `evidence_against`,
  `priors`, `posterior`, `brier_score`. Reviewers must populate evidence on
  both sides — symmetric epistemology is enforced by `mpacr-validation`.
- Validator retries critic outputs that lack `evidence_against` once with a
  sharpened prompt, then escalates to `verdict: "ask_user"`.
- Per-critic timeouts (`mpacr_per_critic_timeout_ms`, default 180 s) prevent
  K critics from sharing one parent wallclock budget.
- Synthesis tolerates `K - quorum` critic skips. Default quorum =
  `max(1, ceil(K * 0.6))` (K=3 → 2, K=5 → 3, K=6 → 4).
- Skipped critics emit a synthetic `verdict: "skipped"` so synthesis can
  proceed without blocking.
- **Brier-score calibration loop** persists per-`(expert_id, workflow)` pairs
  to the new `calibration_record` table. `Calibration.recommendPriorAdjustment`
  returns a recommended shift in `[-0.1, +0.1]` once the sample exceeds
  `MIN_CALIBRATION_SAMPLES` (20).

### 3LMA — Three-Layer Memory Architecture (Stream B)

Adds **semantic** (cross-session domain facts) and **procedural** (successful
task recipes) layers atop the existing episodic memory. All three layers share
the existing `personal_memory_note` table — no new storage backend.

- `MemoryScope` enum extended with `"semantic"` and `"procedural"`.
- `ThreeLayerMemory.recordSemanticFact / recordProceduralRecipe / searchSemantic
  / searchProcedural / detectDomain` — full Effect Service.
- **Memory Consolidator** background process: encode → pattern-extract →
  replay → decay. Triggered manually via `openagt mem consolidate`; full
  scheduler integration is roadmap.
  - Pattern extraction: `(domain, subject, predicate, object)` triples
    repeating ≥ `min_pattern_occurrences` (default 3) become semantic facts
    with `confidence = min(0.95, n / 10)`.
  - Decay: `importance *= 0.5 ^ (age_days / half_life_days)` with
    rehearsal boost capped at +5. Sub-threshold facts get demoted/soft-deleted.
- **Cross-process advisory locks** (`_advisory_lock` table, 5-minute TTL,
  PID + machine-ID guard) prevent multiple `openagt` processes from
  double-consolidating.
- **WAL checkpoint protection**: consolidator calls `walCheckpointTruncate`
  every 100 writes to bound WAL growth.
- `enrichPlanMemory` Effect post-processor injects domain-relevant semantic
  facts and procedural recipes into `plan.memory_context.note_ids` at plan
  creation time — best-effort, falls back to bare plan on backend hiccup.
- `taskSignatureFor({goal, workflow})` produces a stable lookup key
  (`${workflow}:${domain}:${sorted_top3_keywords}`) so a recipe recorded for
  one task is reused for the next.

### DXR + PTS — Dynamic Expert Registry & Prompt Template System (Stream C)

User-extensible expert roles + centralized prompt templates with A/B testing.
**Conservative path: the closed 38-role `CoordinatorNodeRole` enum stays
shut.** User experts attach by inheriting from a builtin role.

- `.opencode/experts/*.md` loader (mirrors agent/command frontmatter pattern).
  Loaded into Config under the new `expert` key during dir-walk.
- `ExpertRegistry` Effect Service — lazy-init reads from Config on first
  query, falls back to builtins when Config is unavailable (test environments).
- `applyExpertOverride(node, entry)` pure helper: replaces prompt + expert_id
  + memory_namespace, **preserves `role` and `output_schema`** so EffortProfile
  dispatch and verdict parsing keep working transparently.
- `applyUserExpertsToPlan(plan)` Effect post-processor: each plan node whose
  role matches a user expert's `inherits` gets the override applied. First-
  registered-wins per parent role.
- **Prompt Templates**: 7 builtin templates extracted from
  `coordinator/*Node()` factories to `coordinator/prompts/<role>/<variant>.md`
  with byte-equal snapshot tests guarding against drift:
  `reviser/{default, no-target}`, `reviewer/checkpoint`, `planner/default`,
  `verifier/shard`, `reducer/{default, project-deep-dive}`.
- **Thompson sampling-lite** picker over the new `prompt_outcome` table:
  `Beta(s+1, f+1)` posterior sampling per variant + 5 % exploration budget.
  Pure (no DB dep) so tests use deterministic PRNG.

### Database & migration safety net

- **`_schema_version`** audit table records every applied migration with
  SHA-256 checksum. Backfilled on first upgrade. `openagt db status` lists
  applied migrations and runs `PRAGMA integrity_check`.
- **`_advisory_lock`** for cross-process coordination (B.3).
- **`calibration_record`** for Brier-score history (A.4).
- **`prompt_outcome`** for per-variant A/B telemetry (C.5).
- All four new migrations are append-only and idempotent.

### CLI surface

| Command | Purpose |
|---|---|
| `openagt db status` | List applied migrations, integrity check |
| `openagt experts list [--source builtin\|user] [--domain X]` | List registered expert roles |
| `openagt experts show <role>` | Show full expert definition (frontmatter + prompt body) |
| `openagt cal show [expert_id] [--since unix-ms]` | Brier score + recommended prior shift per expert |
| `openagt prompt stats [role] [--days 30]` | Per-variant outcome stats from prompt_outcome telemetry |
| `openagt mem consolidate` | Manually run a memory consolidation pass |

All commands support `--format json` for ops automation.

## Migration & Compatibility

- **Zero breaking changes.** Existing configs / sessions / databases load
  unchanged. New schema fields all have safe defaults.
- New role enum entries (`steel-manner`, `red-team-critic`, `defender`,
  `synth-reviser`, `calibrator`) are **appended** to `CoordinatorNodeRole` —
  the closed enum is preserved, only extended.
- `EffortProfile.mpacr_enabled` defaults to `false`, so v1.20 plan shapes are
  byte-identical to v1.21 plan shapes when MPACR is off.
- `MemoryScope` enum extended with `"semantic"` and `"procedural"`. Existing
  notes load unchanged; new scopes only used by ThreeLayerMemory writes.
- `coordinator-personal.test.ts` and `coordinator-runner.test.ts` test
  fixtures were updated to add the new `ThreeLayerMemory` and `ExpertRegistry`
  layers to their `Layer.mergeAll` calls. Downstream test suites that
  construct their own coordinator layer must do the same.

## Hazard register coverage

The 15 hazards documented in the v1.21 implementation plan
(`.claude/plans/potencial-harzard-replicated-ullman.md`) are all mitigated:

| # | Hazard | Mitigation |
|---|---|---|
| H1 | Single critic fail → synthesis blocks forever | A.5 quorum + `skippedVerdict` |
| H2 | K critics share parent wallclock | `mpacr_per_critic_timeout_ms` |
| H3 | `evidence_against` retry budget cascade | A.3 single retry + counted against `max_model_calls` |
| H4 | Models fake evidence to pass schema | A.4 Brier loop + C.5 Thompson sampling auto-downweights |
| H5 | DB migration leaves half-state | B.1 `_schema_version` + `integrityCheck()` |
| H6 | FTS5 trigger drift | B.2 direct trigger test |
| H7 | Pattern extraction creates poisoned facts | B.3 confidence ≤ 0.95 cap + ≥ 0.5 default filter |
| H8 | WAL bloat during consolidation | B.3 `walCheckpointTruncate` every 100 writes |
| H9 | Multi-process double-consolidation | B.3 `_advisory_lock` + 5-min TTL |
| H10 | User expert vs parent schema conflict | C.4 `output_schema` preserved + warn-not-fail guard |
| H11 | Prompt extraction drift | C.3 byte-equal snapshot tests |
| H12 | A/B variant cold-start starvation | C.5 5 % `EXPLORATION_FRACTION` |
| H13 | Outcome attribution wrong | `prompt_outcome` only on verifier-result completion |
| H14 | Bundled prompts lost in build | C.2 startup log "loaded N prompts" |
| H15 | PR doesn't run tests → stream regressions | P0 PR-blocking CI gate |
