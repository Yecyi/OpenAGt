## Summary

OpenAGt v1.20.1 is a patch release for the coordinator and memory consolidation paths reviewed after v1.20.0. It focuses on making runtime behavior match the shipped safety contracts rather than adding new user-facing surface area.

## Fixes

- Render coordinator prompt templates with concrete task context before storing or sending prompts, so planner, verifier, reducer, and reviser templates no longer leak raw `{{placeholder}}` tokens into runtime prompts.
- Enforce MPACR critic quorum in coordinator runtime before synthesis can call a model. If too few critics produced substantive non-skipped verdicts, synthesis records an `ask_user` escalation instead of accepting a possible pass.
- Persist validator-normalized MPACR verdicts into task result metadata, including retry/escalation fixes from malformed or one-sided critic output.
- Record prompt outcome durations from the completed or failed task record instead of the pre-start task snapshot.
- Advance semantic memory timestamps when decay keeps a fact, preventing repeated consolidation runs from applying the same age window again.
- Gate JSON-to-SQLite migration on a dedicated durable migration marker instead of the SQLite database file, so existing databases can still import old JSON storage when the marker is absent.
