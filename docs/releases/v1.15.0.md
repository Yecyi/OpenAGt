## Highlights

- Coordinator Runtime v1: task graph execution now includes dependency ordering, `write_scope`, `read_scope`, acceptance checks, and task priority.
- Personal Agent Core v1: profile, workspace, and session memory are now backed by stable backend primitives, along with inbox and wakeup models.
- Approval and Safety Envelope v1: shell permission flows now emit structured `shell_safety` metadata for server, TUI, and SDK consumers.
- Stable release packaging is now aligned across CLI, server, and the JavaScript SDK.

## New in v1.15.0

- New runtime event surfaces: `coordinator.*`, `inbox.*`, `scheduler.*`, and `memory.updated`
- Expanded task metadata in `task-runtime`: `write_scope`, `read_scope`, `acceptance_checks`, `priority`, and `origin`
- SQLite FTS5-backed personal memory ranking for profile, workspace, and session memory
- Stable shell approval metadata via `shell_safety`
- Stable release workflow, checksum generation, and packaged asset naming for Windows, macOS, and Linux

## Install / Upgrade

- Stable installs are shipped through GitHub Release assets
- Windows users should prefer `OpenAGt-Setup-x64.msi`
- macOS and Linux users should use the matching archive for their platform
- Source builds still require SDK generation:

```bash
bun run --cwd packages/sdk/js script/build.ts
```

## Compatibility / Breaking Notes

- The `opencode` compatibility alias remains available, but stable release artifacts and documentation are now centered on `openagt`
- This release line only covers CLI, server, and the JavaScript SDK
- Flutter is not included in the `v1.15.0` support matrix
- Stable packaging is distributed as archives plus a Windows MSI; npm, Homebrew, and winget are not first-party release channels in this version

## Validation

- `bun typecheck` passes in `packages/openagt`
- `bun run release:verify` passes
- release-critical package-local suites covering CLI smoke, session, tool, agent, security, installation, MCP, provider, PTY, file, snapshot, storage, and supporting runtime modules pass on `release/v1.15.0`
- the single aggregate command `bun test --timeout 30000` remains slower than the current outer timeout budget on this machine, so RC validation is tracked through split suites instead of that single entrypoint

## Known Issues

- Some legacy `opencode`-named scripts and compatibility paths still exist in the repository; they remain supported for compatibility, but `openagt` is the primary release identity
- This release does not ship Flutter client artifacts
- The following tests are still skipped on the RC branch:
  - `tool.bash abort > captures stderr in output`
  - `Worktree > createFromInfo > creates and bootstraps git worktree`
  - `unicode filenames modification and restore`

## Checksums / Assets

- Every published asset is accompanied by `SHA256SUMS.txt`
- Stable asset names are fixed to the following set:
  - `openagt-windows-x64.zip`
  - `OpenAGt-Setup-x64.msi`
  - `openagt-linux-x64.tar.gz`
  - `openagt-macos-arm64.tar.gz`
  - `openagt-macos-x64.tar.gz`
