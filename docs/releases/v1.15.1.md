## Highlights

- Ships the security and CI hardening from PR #4 as a patch release on top of v1.15.0.
- Removes the inherited public website and share surface from the supported release path.
- Keeps release validation strict with generated SDK checks, generated config schemas, audit policy checks, dependency audit, lint, typecheck, and focused regression tests.

## Fixes

- Hardens MCP OAuth callback rendering, state handling, redirect URI validation, port conflict behavior, transport cleanup, and timeout budgeting.
- Replaces terminal WebSocket long-lived credential URLs with short-lived PTY tickets while keeping legacy compatibility.
- Restores scoped provider and remote config environment handling so credentials do not stay in global `process.env`.
- Validates auth content loaded from environment with the same schema path as disk auth.
- Writes secret JSON files through atomic same-directory temp files with explicit file modes.
- Tightens read and edit tooling around oversized binary attachments, large log reads, exact overwrite requests, and whitespace-preserving file reads.
- Removes stale public share APIs and generated SDK surface for those endpoints.
- Resolves SolidStart audit findings by using the vendored workspace package and regenerated lockfile.
