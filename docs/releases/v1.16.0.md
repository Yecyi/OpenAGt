## Highlights

- OpenAGt v1.16.0 focuses on release trust, safety metadata stability, and runtime diagnostics.
- Windows GA releases now require signing configuration; unsigned builds remain RC-only by default.
- `shell_safety` is versioned so runtime, server, TUI, and SDK consumers can validate the same envelope.
- `openagt debug doctor` and `openagt debug bundle` provide first-party diagnostics for support and bug reports.
- Coordinator and Personal Agent contracts are tightened for graph projection, inbox state, and Flutter control-panel readiness.

## Key Features

- Signed-GA / unsigned-RC release policy
- SBOM and checksum release assets
- Packaged binary smoke tests in release workflow
- `shell_safety.version = 1`
- `openagt debug doctor`
- `openagt debug bundle --session <id>`
- SSE envelopes with `schema_version`, `event_id`, `trace_id`, and `timestamp`
- SDK helpers for shell safety, coordinator projections, and inbox overview

## Compatibility Notes

- Existing `opencode` aliases and `.opencode` config compatibility remain.
- Existing inbox rows using older states are normalized at read time.
- Flutter is still not part of the stable release artifact set.
- Windows SmartScreen may still warn on unsigned RC assets.

## Validation

- `bun typecheck` from `packages/openagt`
- focused security tests for shell safety and high-risk command handling
- focused agent tests for Coordinator and Personal Agent behavior
- release workflow smoke tests against packaged binaries
